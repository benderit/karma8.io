10. На MySQL выросла нагрузка, сервера не хватает. Что делать?

0. Найти узкие места
	-Найти бутылочное горлышко в системе и решить проблему с ним, дальше по обстоятельствам.
1. Вертикальное маштабирование
	-Увеличить мощность сервера
2. Горизонтальное масштабирование
	-Увеличить число серверов
	-Использовать балансировщик нагрузки
	-Использовать оркестровщик типа k8s
3. Анализ конфигурации сервисов(nginx, mysql, etc)
	-Возможно можно затюнить, например поместить всю базу в ОЗУ, или дропать транзакции раз в секунду, а не по одной
3. Оптимизировать код приложения
	-По максимуму убрать взаимоблокировки в СУБД
	-Использовать разные базы даннных под разные задачи
	-Использовать несколько баз данных в рамках одной СУБД (MySQL), даст гор. масштабирование (шардирование)
	-Убрать лишние циклы, запросы и так далее
4. Улучшить работу с БД
	-Анализ длинных запросов к СУБД
	-Тюннинг конфигурации	
	-Распределенные вычисление (оно же гор. масштабирование)
	 MySQL можно заменить на MariaDB Galera + MaxScale
	 операции на чтение точно станут быстрее за счет балансировки, мультимастер абсолютно консистентен и 
	 с любого мастера можно получить актульные данные, в отличие от master-slave.
	 на запись кластер галеры прироста не даст.
	-Оптимизировать на уровне программного обеспечения, например использовать redis для работы с сессиями,
	использовать кеширование, CDN, ClickHouse для аналитических запросов (статистика например)
	То есть определить верно ли выбраны средства для работы, там где нужно быстро - нужно быстрое решение
	там где нужно надежно - надежное.
